Yes — Pyodide.

It runs Python entirely in the browser via WebAssembly. No backend needed at all.

    User's code executes on their own machine in the browser
    You just deploy the frontend to Vercel — 100% free, forever
    No security risk to you since nothing runs on your server

The tradeoff: you'd replace your FastAPI backend + sys.settrace with Pyodide's Python runtime running in a Web Worker, and reimplement the tracer in pure Python that Pyodide executes.

Python Tutor actually does exactly this — it's the standard approach for browser-based Python debuggers.
